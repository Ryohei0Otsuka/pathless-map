# Pathless Map

**Pathless Map** は、ファイルの流れを「実名なし」で整理するための導線マップです。

実際の端末名、ファイル名、フォルダパス、顧客名などは入力せず、  
「取得元」「作業場所」「処理」「確認」「判定」「格納先」などの抽象パーツを使って、ファイルがどこから来て、どこを通り、どこへ格納されるのかを見える形にします。

> 機密を入れないファイル導線整理ツール

## 何のためのツールか

複数の端末、チャット、クラウド、作業場所をまたぐファイル操作は、文章だけでは流れを把握しづらいことがあります。

特に、以下のような工程が重なると、手順そのものよりも「ファイルが今どこにあり、次にどこへ行くのか」が分かりにくくなります。

- 取得
- 移動
- 圧縮
- 解凍
- 加工
- リネーム
- 確認
- 判定
- 格納
- 戻し
- 保留

Pathless Map は、その流れをパーツと矢印で整理するためのデモアプリです。

手順書の代わりではなく、  
手順を理解するための「地図」として使うことを目的としています。

## なぜ作ったのか

ファイル操作の手順は、ひとつひとつを見ると単純でも、  
実際には「どこから来て、どこで何をして、どこへ置くのか」が複雑になりがちです。

さらに、1つのタスクで扱うファイルが1つだけとは限りません。

同じタスクの中でも、ファイル種別ごとに取得元や加工内容、確認ポイント、格納先が違うことがあります。

その流れを頭の中だけで覚えようとすると、抜けや勘違いが起きやすくなります。

そこで、具体的な業務情報を使わずに、  
タスクごと・ファイルごとの流れだけを抽象化して見えるようにするために作りました。

## 機密情報について

このアプリは、機密情報を入力しない前提で作っています。

以下のような情報は入力しないでください。

- 実際の端末名
- 実際のファイル名
- 実際のフォルダパス
- サーバー名
- URL
- 顧客名
- 個人名
- Slack の実チャンネル名
- SharePoint の実サイト名
- 業務固有の識別子

入力する場合は、必ず抽象化してください。

例：

- 端末A
- 作業場所X
- ファイル種別A
- 共有クラウド
- 確認ポイント
- 最終格納先

## 機密は守られるのか

このアプリは、外部DBを使わないローカル前提のデモです。  
ただし、「ローカルだから安全」とは考えていません。

機密を守るために、次の考え方を取っています。

- 実名を入力しない
- 実パスを入力しない
- 実ファイル名を入力しない
- 業務固有の名前を入力しない
- 抽象化した名前だけで整理する
- 起動時に注意喚起を表示する
- 機密情報らしい文字列が入った場合は警告する
- 保存しないモードを用意する
- 保存する場合も、入力内容は抽象名だけにする

このツールが機密を自動的に守るのではなく、  
**機密を入れない使い方を前提に、流れだけを整理する**ためのものです。

## できること

- タスクごとに導線マップを分ける
- 1つのタスク内で複数ファイルの流れを分ける
- パーツを置いてファイルの流れを作る
- 矢印で工程をつなぐ
- 判定カードから OK / NG / 保留 の流れを作る
- パーツごとにメモを書く
- 矢印ごとに工程名や補足を書く
- 抽象名だけで流れを整理する
- スマホでも見やすい形で確認する

## 使いどころ

- 複雑なファイル操作の流れを整理したいとき
- 手順を文章だけで覚えるのが難しいとき
- 1タスク内で複数ファイルを扱うとき
- ファイルごとにルートや加工工程が違うとき
- 作業の全体像を確認したいとき
- 認識合わせのために、抽象化した図を見せたいとき
- 「どこで何をするのか」を地図として確認したいとき

## 注意

Pathless Map は、実際の業務情報・機密情報・個人情報を保存するためのツールではありません。

このアプリに入力する内容は、必ず抽象化してください。

具体名ではなく、流れを見る。  
実パスではなく、導線を見る。  
ファイル名ではなく、役割を見る。

それが Pathless Map の使い方です。
